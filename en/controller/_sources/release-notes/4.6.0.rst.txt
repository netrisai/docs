Netris |version| (Feb/10/2026)
==================================

What's new in Netris |version|?
-------------------------------

Features
--------
.. New capabilities visible to customers. One line each; optional 1â€“2 lines of context.

- **EVPN-on-Host. Linux Host VTEP.** Any Ubuntu 24.04 host can be configured as an EVPN VTEP, therefore extending the VXLAN overlay into the host.
- **NVLink Multi-Node.** Netris can now create NVLink partitions in NVL72 and NVL144 equipped systems.
- **eBGP imports non-default routes.** The operator may now apply a BGP community 0:7 to signal Netris to import any non 0.0.0.0/0 eBGP prefixes into the VPC.
- **Control plane verification of VTEP reachability.** Netris managed VTEPs (switches and Softgates) keep track of the reachability of other Netris VTEPs.
- **CloudStack supported with Cumulus Linux.** CloudStack now integrates with Nvidia switch fabrics running Cumulus Linux.
- **VPC peering is now supported in CloudStack**.
- **Spectrum-X Deployment Guide v1.3 support** Spectrum-X Deployment Guide v1.3 is now supported.
- **SNMPv2.** Netris can now configure SNMPv2 read-only access on supported switches
- **Topology View Filtering.** Topology view now supports filtering by device name.
- **Dell/BC-SONiC VPC peering.** VPC peering can now be established on :ref:`Dell/BC-SONiC Ethernet <overlay-network-functions>` fabrics.
- **Dell/BC-SONiC L3VPN.** L3VPN is now supported on :ref:`Dell/BC-SONiC Ethernet <overlay-network-functions>` fabrics.

Enhancements
------------
.. Customer-facing enhancements to existing functionality and features. If internal-only, keep wording generic.

- **XDP dataplane acceleration for SoftGate HS**.
- **DHCP Relay** now supports inter-VPC routing on all :ref:`supported-switch-hardware`.
- **ACLs on Arista EOS** are now available on :ref:`supported-switch-hardware`.
- **Topology validation** added new fields in custom JSON.
- **Added switch port health checks** Switch port healt checks now include Bit Error Rate (BER) and temperature monitoring on NVUE-supported platforms.
- **Performance and UI improvements.**

  - Add Network > Links page showing all topology links.
  - Tags are now Labels to remove confusion with VLAN tags.
  - Added NIC stats for Generic/agentless server type derived from switch ports.
  - Improved database scalability in larger deployments.
  - Server Cluster UI now displays members in alphabetical order.
  - Rename Topology Export Full to Cut Sheet.
  - CloudStack agent automatically generates ACLs for intra-subnet communication.
  - Improved collectd optical values export.
  - System ASN range is now editable.
  - Can now show more than 4 tx/rx optical signals in Optical Graph.
  - In some CloudStack deployments enable and enforce IP forwarding property in Linux kernel.

- Fixed backend crash after creating LB service.
- Fixed the inability to edit NAT rules when a VPC filter is applied.
- Fixed Inventory Filtering failing under certain conditions.
- Fixed Inventory API filtering by label, sites, types, health, and custom.
- Fixed filtering of ports with unknown states under certain conditions.
- Fixed NVUE check_signal monitor false positives on optical switch ports.
- Fixed ACL creation time stamp.
- Fixed RX values in graphs.
- Fixed filter reset button in ACL listing
- Fixed NVUE agent restarting, if the previous cycle takes too long.
- Fixed check_signal incorrectly identifying optical ports.
- Fixed signal level reporting on 8 lane optics.
- Fixed incorrectly displaying packet per second values for the RX lines.
- Fixed custom rules in inventory profile support for any port
- Fixed static routes applying to all switches when ApplyTo filter is used.
- Fixed ACLs sometimes filtering even if no filter is applied.
- Fixed ACL creation time stamp.
- Fixed installing reverse rule in NVUE when tcp-establiashed flag disabled.
- Fixed controller hanging when malformed create L4LB API request is received.
- Fixed CPU utilization graph showing incorrect data.
- Fixed ACL functionality when used with SNAT.
- Fixed SoftGate HS DNAT with Port Groups.
- Fixed SNAT is not working when traffic is destined to a host behind DNAT rule.
- Fixed NHN impacting North-South traffic when configuring East-West host networking.
- Fixed Topology>Interface>Details showing incorrect optical graph data.
- Fixed VPC peering with L3VPN Vnets on NVUE platforms.
