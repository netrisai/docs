Netris |version| Release Notes (Jun/15/2026)
============================================

What's new in Netris |version|?
-------------------------------

Features
--------
.. New capabilities visible to customers. One line each; optional 1–2 lines of context.

- **Per-Port FEC Configuration**. Allows operators to configure Forward Error Correction (FEC) mode per physical port via the Netris Controller UI and API. Supported on Cumulus, SONiC, and Arista platforms.

Enhancements
------------
.. Customer-facing enhancements to existing functionality and features. If internal-only, keep wording generic.

- **Host Networking Plugin**. Adds native IPv6 support for tenant Kubernetes clusters, enabling IPv6-only deployments.
- **Topology Rendering Performance**. Optimized rendering of the topology view so large fabrics load and update significantly faster in the UI.
- **Network Links Section Performance**. The Network Links section in the controller dashboard now loads much faster, with improved status-filter logic.
- **Network Interfaces Loading Performance**. Reduced load time of the Network Interfaces view on large-scale fabrics.
- **Graphite → VictoriaMetrics Data Migration on Upgrade**. The controller upgrade flow now automatically backs up the existing Graphite dataset, transforms it for VictoriaMetrics, imports it, and validates integrity, preserving historical metrics continuity.
- **InfiniBand Debug Messages**. Updated debug-mode log messages for the InfiniBand (IB) integration to be clearer and more actionable when troubleshooting.

Bug fixes
---------
.. Customer-facing defect fixes. If internal-only, keep wording generic.

- Fixed Cumulus agent entering a loop when a route-map was created without a match rule; empty route-map sequences are now skipped safely.
- Fixed DNAT not working when backend IPs configured under the L4LB used the same ports as DNAT rules.
- Fixed backend health checks not responding within 1 second under load in large-scale deployments.
- Fixed Terraform silently accepting undefined Fabric Type values (e.g. "ew-plane5"), which resulted in a blank Fabric Type in the controller.
- Fixed wrong prefix length in the prefix-list used for Cumulus VPC peering; the agent now configures ``min-prefix-len``.
- Fixed a UI error when changing the eBGP local IP for a peer created with the "connect with vnet" option.

Security fixes
----------------

No security fixes in this release.
