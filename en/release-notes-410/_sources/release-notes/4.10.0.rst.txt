Netris |version| Release Notes (May/15/2026)
==============================================
 
What's new in Netris |version|?
-------------------------------
 
Features
--------
 
- **Inventory Switch Role**. Adds a new Switch Role property (generic, leaf, spine, super-spine) to inventory switch objects, enabling operators to explicitly define each switch's role in the fabric topology. Existing switches are automatically assigned a role based on their location naming convention during the upgrade process. (NET-345, NET-346, NET-347)
- **Inventory Profile: Fabric Type**. Adds a new Fabric Type property to Inventory Profiles, allowing administrators to define the fabric tier and traffic directionality for each profile. Supported types include east-west, north-south, out-of-band, and general-purpose. (NET-348, NET-349, NET-350)
- **Inventory Profile: Reference Architecture**. Adds a new Reference Architecture selector to Inventory Profiles, allowing administrators to associate a profile with a recommended deployment pattern. (NET-337, NET-341)
 
Enhancements
------------
 
- **Server Cluster Scalability**. Improved the server cluster add/edit API to support creating clusters with a large number of servers (2000+) in a single operation. (NET-354)
- **Swagger API Documentation**. Updated the Swagger API definitions with all missing API endpoints. (NET-373)
 
Bug fixes
---------
 
- Fixed L4LB not working when 1:1 DNAT rules are configured for the same backend IPs. (NET-378)
- Fixed switch model not displaying correctly in the controller inventory for Cumulus Linux switches. (NET-334)
- Fixed IPv4 and IPv6 addresses displaying concatenated without proper separation in the UI Network Links view. (NET-320)
- Fixed IPv6 VPC peering not working as expected in multi-cluster topologies, a regression introduced in 4.9.0. (NET-315)
- Fixed CloudStack agent not cleaning up VRF interfaces after the last VM in a VRF VNet is migrated or deleted. (NET-258)
- Fixed suboptimal routing with VPC peering and EVPN Multihoming on Cumulus by implementing proper local leaking logic. (NET-220)
 
Security fixes
----------------
 
No security fixes in this release.
