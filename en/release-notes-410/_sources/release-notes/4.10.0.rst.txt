Netris |version| Release Notes (May/15/2026)
=============================================

What's new in Netris |version|?
-------------------------------

Features
--------

- **Inventory Switch Role**. Switch objects in the inventory now include a role property (leaf, spine, super-spine, or generic), enabling operators to explicitly classify switches by their position in the fabric hierarchy. The role is auto-populated based on the switch location during migration. (NET-345)

- **Inventory Profile: Fabric Type Property**. Inventory profiles now support a fabric type property that defines the traffic directionality and fabric tier the profile is designed for — including east-west (including multi-plane configurations), north-south (border/edge), out-of-band management, and general-purpose roles. (NET-348)

- **Inventory Profile: Reference Architecture**. Inventory profiles now support a reference architecture property that defines the recommended deployment pattern for the profile. (NET-337)

Enhancements
------------

- **UFM reconciliation controls via Global Settings**. Dry-run mode, debug mode, and the reconcile interval can now be controlled through the controller's global settings. (NET-267)

Bug fixes
---------

- Fixed suboptimal routing with VPC peering and EVPN Multihoming on Cumulus Linux, including corrections to LINKLOCAL-V6 prefix list handling in remote leaking scenarios and proper remote IPv6 address advertisement via LLDP TLV. (NET-220)

- Fixed suboptimal routing with VPC peering and EVPN Multihoming on Dell SONiC platforms. (NET-299)

- Fixed an issue where the CloudStack agent did not delete the VRF interface after the last VM from that VRF VNet was migrated or deleted. (NET-258)

- Fixed a regression where IPv6 VPC peering was not working as expected on Cumulus Linux 5.16. (NET-315)

- Fixed a UI display issue in the Network Links section where IPv4 and IPv6 addresses were rendered without proper separation. (NET-320)

- Fixed an issue where L4LB did not work when 1:1 DNAT rules (or other NAT rules) were configured for the same backend servers. (NET-378)

Security fixes
--------------

No security fixes in this release.
